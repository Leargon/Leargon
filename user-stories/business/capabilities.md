#### USER STORY 'Capability map'
**AS A** company\
**IF** \
**I WANT** to define and organise business capabilities in a hierarchical map\
**SO THAT** we can document what the organisation is able to do, independent of how processes or teams are structured

#### USER STORY 'Actualisation'
**AS AN** admin\
**IF** \
**I WANT** to administer a capability's name, description, and owning organisational unit\
**SO THAT** all information about the capability is up-to-date and the responsible team is documented

#### USER STORY 'Decomposition'
**AS AN** admin\
**IF** \
**I WANT** to create sub-capabilities under a parent capability\
**SO THAT** I can decompose high-level capabilities into more granular building blocks

#### USER STORY 'Strategic alignment'
**AS AN** admin\
**IF** \
**I WANT** to classify capabilities by strategic importance (Differentiating, Essential, Commodity) and maturity level\
**SO THAT** the capability map can be used to guide investment decisions and identify gaps

#### USER STORY 'Link to processes'
**AS AN** admin\
**IF** \
**I WANT** to link capabilities to the business processes that realise them\
**SO THAT** the connection between what the organisation can do and how it does it is explicitly documented
