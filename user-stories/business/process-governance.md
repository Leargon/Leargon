#### USER STORY 'Basic'
**AS A** company\
**IF** \
**I WANT** to define business processes and assign a specific user as process owner\
**SO THAT** we can develop process governance and model the process flow

#### USER STORY 'Actualisation'
**AS A** process owner or admin\
**IF** \
**I WANT** to administer the process's name, description, and classifications to reflect the reality\
**SO THAT** all information of the process is up-to-date

#### USER STORY 'Modeling'
**AS A** process owner or admin\
**IF** \
**I WANT** to model the process by creating a task or subprocess, or link an existing task or subprocess, use gateways for decisions (XOR, OR, AND), and events (start, intermediate, end)\
**SO THAT** everyone can check the details of the process 

#### USER STORY 'Child process'
**AS A** process owner or admin\
**IF** \
**I WANT** to create child processes\
**SO THAT** I can define more details

#### USER STORY 'Business domains'
**AS A** process owner or admin\
**IF** \
**I WANT** to assign the business process to a domain\
**SO THAT** I can group the business process by domain

#### USER STORY 'Executional units'
**AS A** process owner or admin\
**IF** \
**I WANT** to assign executional unit to the business process\
**SO THAT** I can specify the operational unit, which does execute this process

#### USER STORY 'Parent process'
**AS A** process owner or admin\
**IF** \
**I WANT** to assign a parent process\
**SO THAT** this business process is getting a child process of the selected parent process
