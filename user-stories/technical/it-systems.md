#### USER STORY 'Create IT system'
**AS AN** admin\
**IF** \
**I WANT** to create an IT system by providing a name in at least one supported locale, its type, and optionally a vendor name and system URL\
**SO THAT** the system is registered in the IT landscape and can be linked to processes and organisational units

#### USER STORY 'List IT systems'
**AS A** logged in user\
**IF** \
**I WANT** to see a list of all IT systems with their names, types, and vendor information\
**SO THAT** I get an overview of the organisation's IT landscape

#### USER STORY 'View IT system details'
**AS A** logged in user\
**IF** \
**I WANT** to see all details of an IT system including names, descriptions, type, vendor, system URL, owning organisational unit, and linked processes\
**SO THAT** I have complete information about that system

#### USER STORY 'Update IT system names'
**AS AN** admin\
**IF** \
**I WANT** to update the names of an IT system across all supported locales\
**SO THAT** names are correct and up-to-date in every language

#### USER STORY 'Update IT system descriptions'
**AS AN** admin\
**IF** \
**I WANT** to update the descriptions of an IT system across all supported locales\
**SO THAT** descriptions are correct and up-to-date in every language

#### USER STORY 'Update IT system type'
**AS AN** admin\
**IF** \
**I WANT** to set the type of an IT system (SaaS, On-premise, Custom-built, PaaS)\
**SO THAT** the deployment model of the system is clearly specified

#### USER STORY 'Update IT system vendor'
**AS AN** admin\
**IF** \
**I WANT** to update the vendor or provider name of an IT system\
**SO THAT** the supplier relationship is documented

#### USER STORY 'Update IT system URL'
**AS AN** admin\
**IF** \
**I WANT** to update the system URL of an IT system\
**SO THAT** users can navigate directly to the system

#### USER STORY 'Update IT system owning unit'
**AS AN** admin\
**IF** the organisational unit exists\
**I WANT** to assign or change the owning organisational unit of an IT system\
**SO THAT** the team responsible for operating this system is documented

#### USER STORY 'Link IT system to process'
**AS A** process owner or admin\
**IF** the IT system exists\
**I WANT** to link an IT system to a business process\
**SO THAT** it is documented which system supports the execution of that process

#### USER STORY 'Unlink IT system from process'
**AS A** process owner or admin\
**IF** the IT system is currently linked to the process\
**I WANT** to remove the link between an IT system and a business process\
**SO THAT** the relationship is removed when no longer accurate

#### USER STORY 'Delete IT system'
**AS AN** admin\
**IF** \
**I WANT** to permanently delete an IT system\
**SO THAT** decommissioned or incorrectly created systems are removed from the inventory
